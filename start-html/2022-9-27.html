规划一个简单的网站

非常便利但容易被滥用，因为div没有语义值，尽量在没有更好语义方案才用div。否则文档升级维护非常麻烦。

web信息架构，组织一个站点信息、功能，让它更易找、易用。

在完成页面内容的规划后，一般应按部就班地规划整个网站的内容，要可能带给用户最好的体验，需要哪些页面、如何排列组合这些页面、如何互相链接等问题不可忽略。这些工作称为信息架构

规划设计步骤

  1. 列出对所有页面通用的内容，比如导航菜单、页脚、联系方式etc
  2. 绘制结构草图，记录每一块的作用
  3. 头脑风暴出要加入站点的功能内容
  4. 对上面列出的功能进行分组，组合看哪些页面可以放在同一个页面
  5. 绘制站点草图，一个气泡一个页面，连线表示工作流

参考MDN：规划一个简单的网站 中顺序，开始设计。

HTML 之所以以宽松的方式进行解析，是因为 Web 创建的初心就是：人人可发布内容，不去纠结代码语法。如果 Web 以严格的风格起步，也许就不会像今天这样流行了。
